/ 04 · Shopify App — Checkout Extensibility

"Order before 14:00,
get it Tuesday."

PDP, sepet, checkout ve thank-you sayfasında dinamik teslimat tarihi. Shipping rule, cut-off, hafta sonu, tatil ve ülke-spesifik logic. Multi-language, multi-country, Checkout Extensibility uyumlu.

Type
Public Shopify App
Stack
Node · React · Liquid
Surface
4 storefront areas
Tier
Free / Pro / Advanced
Problem

Shoppers want clarity, not "1-3 business days".

Cart abandonment'in temel sebeplerinden biri belirsiz teslimat tarihi. "Order in next 4h 12m, get it Tuesday" gibi spesifik commit, Amazon'un kullanıcı eğitiminden geliyor — kullanıcı buna alıştı. Ama Shopify mağazaları çoğu zaman "1-3 business days" diyor: çünkü gerçek tarih hesabı complex (cut-off, weekend, holiday, country-specific).

ETAly bunu çözer: her mağazaya özel rule engine + tek satır code yok kurulum + checkout'a kadar consistent badge.

Solution

Rule engine + checkout extensibility + multi-country.

Stack: Node + Express backend, Shopify Polaris admin, Liquid + Checkout Extensions.

  • Rule engine — shipping zone × product tag × variant × cut-off time × weekend × holiday calendar
  • Product / collection / tag / variant targeting — granüler kontrol
  • 4 surface — PDP, cart drawer, checkout (Extensibility), thank-you
  • Multi-language — admin'den çeviri matrisi (EN, DE, TR, ES, FR)
  • Multi-country — IP geo + Shopify Markets entegrasyonu
  • Analytics dashboard — conversion lift ölçümü
  • Subscription billing — Free / Pro / Advanced tier (Shopify Billing API)

Checkout Extensibility için UI Extension yazdım: Shopify'ın checkout UI extension framework'üyle native render (iframe değil), tüm Plus mağazalar destekli.

Stack

Modern Shopify app stack.

  • Node + Express
  • Polaris admin UI
  • Checkout UI Extensions
  • Liquid PDP/cart blocks
  • Shopify Markets API
  • Holiday calendar (i18n)

/ 05 · Integration — Bi-directional Sync

Shopify ⇄ Shopware · two-way.

Shopify ile Shopware 5 ve 6 arasında ürün, sipariş, müşteri ve stok'un bi-directional senkronizasyonu. Order status + payment state + fulfillment sync logic'i ve retry mekanizmasıyla beraber. Manuel ops'u büyük ölçüde azalttı.

Type
ERP Integration
Versions
SW 5 + SW 6
Sync
Bi-directional
Stack
Laravel · REST · MySQL
Problem

İki platform, iki gerçeklik, sıfır otomasyon.

Shopify storefront + Shopware ERP — iki platform, her biri kendi product/order veri modeline sahip. Müşteriler Shopify'da satın alıyor ama operasyon Shopware'da. Stok değişikliği iki taraf da güncellemek gerekiyordu, ve Shopware 5 ile 6 farklı API'lere sahip (5: REST + older auth, 6: REST + new auth + plugin store).

Yarısı manuel yapıldığında: stok tutarsızlıkları, sipariş gecikmeleri, fulfillment hataları başlıyordu.

Solution

Adapter pattern + mapping layer + retry.

İki versiyonu desteklemek için adapter pattern: tek bir ShopwareClient interface, altında v5 ve v6 implementasyonları. Üst katman versiyondan bağımsız.

  • Bi-directional product sync — name, price, description, variant, stock
  • Order sync — Shopify order → Shopware order (status + payment state mapping)
  • Customer sync — yeni müşteri her iki sisteme yansır
  • Inventory sync — gerçek zamanlı (webhook + saatlik reconciliation)
  • Mapping layer — field name + taxation + currency normalize
  • Retry mekanizması — exponential backoff, 5 deneme, then Slack alert
  • Error log — her transaction MySQL'de indexed

Stack

PHP middleware.

  • Laravel queue + jobs
  • Adapter pattern (SW5 + SW6)
  • Shopify Admin API (GraphQL + REST)
  • Shopware REST API
  • MySQL event log
  • Slack webhook alerts

Outcomes

Manuel ops dramatik düştü.

2-waySync direction
2SW versions
5×Retry on fail
99%Sync success rate
